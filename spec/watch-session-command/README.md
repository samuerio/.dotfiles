## 使用方法

### 1. 启动 pi

```bash
pi
```

### 2. 使用 /watch 命令

```bash
/watch
```

### 3. 选择要监听的会话

会显示可用会话列表，格式为：
- 工作目录
- 时间戳（最近修改时间）

### 4. 停止监听

按 `Ctrl+C` 停止监看

## 功能特性

### ✅ 已实现

1. **列出会话**
   - 扫描 `~/.pi/agent/sessions/` 目录
   - 读取每个 `.jsonl` 文件的第一行获取 header
   - 解析 `cwd` 和 `timestamp`
   - 按修改时间排序（最新的在前）

2. **选择会话**
   - 使用 `ctx.ui.select()` 显示会话列表
   - 格式：`<cwd> (<时间>)`

3. **监听文件**
   - 使用 `fs.watch()` 监听选中的会话文件
   - 记录最后读取的行数
   - 只读取新增的内容

4. **解析和显示**
   - 解析 JSONL 格式的消息
   - 格式化显示：
     - User: 👤
     - Assistant: 🤖
     - Tool Result: 🔧 [toolName]
     - Bash Execution: 💻 $
   - 使用 `ctx.ui.notify()` 显示

### 📝 技术细节

- **会话文件格式**: JSONL（每行一个 JSON 对象）
- **消息类型**: 只处理 `type: "message"` 的条目
- **内容截断**: 每条消息最多显示 200 字符
- **退出机制**: Ctrl+C 终止监听

## 使用场景示例

### 场景 1: 监听后台任务

```bash
# 终端 1: 启动后台 pi 任务
pi '执行一个长时间的任务'

# 终端 2: 监听第一个终端
pi
/watch
# 选择对应的会话
```

### 场景 2: 监听另一个会话

```bash
# 终端 1: 正常使用 pi
pi

# 终端 2: 监听第一个终端的活动
pi
/watch
```

## 局限性

1. **只能看到已写入的内容**: 如果后台进程还没写入文件，监看者看不到
2. **轮询延迟**: 文件变化有轻微延迟（依赖 fs.watch）
3. **无交互能力**: 只能被动查看，不能干预后台会话
4. **消息截断**: 长消息只显示前 200 字符

## 替代方案

如果 `/watch` 不满足需求，可以使用：

### 方案 1: 使用 tmux（pi 官方推荐）

```bash
# 在 tmux 中启动后台 pi
tmux new-session -d -s pi-bg "pi '执行任务...'"

# 在另一个终端监听该会话
tmux attach-session -t pi-bg
```

### 方案 2: 文件重定向 + tail

```bash
# 后台执行并重定向输出
pi '任务' > /tmp/pi-output.log 2>&1 &

# 监听日志文件
tail -f /tmp/pi-output.log
```
